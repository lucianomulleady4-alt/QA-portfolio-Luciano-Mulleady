# Manual Test Cases Documentation

## Test Case 1: User Login
- **Objective:** Verify that users can log in with valid credentials.
- **Preconditions:** User must be registered.
- **Test Steps:**
  1. Navigate to the login page.
  2. Enter valid username and password.
  3. Click the "Login" button.
- **Expected Result:** User should be redirected to the dashboard.
- **Actual Result:**  
- **Status:** Pass/Fail

## Test Case 2: User Registration
- **Objective:** Verify that new users can register successfully.
- **Preconditions:** None.
- **Test Steps:**
  1. Navigate to the registration page.
  2. Fill in required fields (username, password, email).
  3. Click the "Register" button.
- **Expected Result:** User should receive confirmation of successful registration.
- **Actual Result:**  
- **Status:** Pass/Fail

## Test Case 3: Password Reset
- **Objective:** Verify that users can reset their password.
- **Preconditions:** User must have an account.
- **Test Steps:**
  1. Navigate to the login page.
  2. Click on "Forgot Password?"
  3. Enter registered email.
  4. Click the "Submit" button.
- **Expected Result:** User should receive an email with password reset instructions.
- **Actual Result:**  
- **Status:** Pass/Fail
